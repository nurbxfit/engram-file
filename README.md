# engram-file

File-based memory store for AI agents. Persistent, structured, and token-conscious.

```
npx skills add nurbxfit/engram-file
```

Engram gives AI agents a durable memory across sessions using flat files ŌĆö no database, no external service. Entries are structured with types, tags, and module scoping. A Python CLI handles querying, adding, archiving, and validation.

---

## Installation

### Option 1 ŌĆö OpenCode Skill (recommended for AI agents)

```bash
npx skills add nurbxfit/engram-file
```

Installs the `memory` skill into your agent's skill directory. The skill tells the agent how to bootstrap the project ŌĆö download the CLI from GitHub and scaffold the memory directory.

### Option 2 ŌĆö Git Clone

```bash
git clone git@github.com:nurbxfit/engram-file.git
cd engram-file
python scripts/engram.py init   # scaffold memory/ and seed index
```

### Option 3 ŌĆö Manual Bootstrap (from skill instructions)

From an empty project:

```bash
mkdir -p scripts
curl -fsSL https://raw.githubusercontent.com/nurbxfit/engram-file/main/scripts/engram.py -o scripts/engram.py
chmod +x scripts/engram.py
python scripts/engram.py init --pull-skills
```

This installs the CLI and downloads the latest skill file in one step.

## Quick Start

```bash
# Seed the memory index
python scripts/engram.py init

# Save a decision
python scripts/engram.py add \
  --type decision \
  --topic jwt-auth \
  --tags auth,security \
  --note "Use short-lived JWT, avoid sessions" \
  --module auth

# Save an error pattern
python scripts/engram.py add \
  --type error \
  --topic circular-dep \
  --tags db,imports \
  --note "Circular dependency when importing models from __init__.py" \
  --module db

# Query by topic
python scripts/engram.py query --topic jwt

# Filter by module
python scripts/engram.py query --module auth

# Session-start digest
python scripts/engram.py digest --task "implement authentication"

# Archive a superseded entry
python scripts/engram.py archive --topic old-sessions

# Validate integrity
python scripts/engram.py validate
```

## CLI Reference

| Command | Description |
|---------|-------------|
| `init [--pull-skills]` | Scaffold `memory/` directory and seed `index.jsonl` |
| `add` | Create a new entry with schema validation |
| `query` | Search active entries by topic, tags, type, or module |
| `digest` | Print a session-start summary filtered by task hint |
| `archive` | Move an entry to `archive.jsonl` with status change |
| `validate` | Check `index.jsonl` and `archive.jsonl` for schema errors |
| `sync-views` | Regenerate `by-module/` filtered views from the index |

Global options:

| Flag | Env Variable | Description |
|------|-------------|-------------|
| `--root <dir>` | `ENGRAM_PROJECT_ROOT` | Project root directory (default: CWD) |

## How It Works

### Two-Layer Storage

| Layer | File | Cost | When |
|-------|------|------|------|
| Index | `index.jsonl` | Cheap (scanned every session) | Session start |
| Details | `details/<module>/*.md` | Expensive (loaded on demand) | Only when relevant |

The index is a JSONL file (one JSON object per line). Each entry has a `type`, `topic`, `tags`, `note`, and an optional `details` field pointing to a markdown file for deeper context.

### Entry Types

- `decision` ŌĆö Architectural choices
- `error` ŌĆö Bug patterns found and resolved
- `pattern` ŌĆö Reusable approaches
- `convention` ŌĆö Style/project rules

### Symmetrical Bootstrap

The skill file and the CLI are bootstrapped from the same GitHub repository ŌĆö neither hardcodes the other's content:

```
Skill (SKILL.md)                CLI (engram.py)
      Ōöé                             Ōöé
      Ōöé "download engram.py"        Ōöé "download SKILL.md"
      Ōöé from GitHub                 Ōöé from GitHub
      Ō¢╝                             Ō¢╝
      ŌööŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌ¢║ repo ŌŚäŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöś
```

- The skill tells the AI: *"If `scripts/engram.py` doesn't exist, download it from $URL"*
- The CLI can be run with `init --pull-skills` to download the latest skill file

## Development

```bash

git checkout -b feature/your-idea

# Make changes, then:
```

Open a pull request on GitHub. Changes merged to `main` are immediately available to `npx skills add nurbxfit/engram-file` and the raw GitHub URLs used by the bootstrap flow.
